O meu objetivo atual é integrar o ecossistema "PaperClip" (5 agentes de IA operacionais rodando no DeepSeek) utilizando n8n (orquestrador de background) e Evolution API (Gateway de WhatsApp) de forma totalmente embutida neste mesmo repositório (Padrão Monorepo), preparando a arquitetura para ser futuramente implantada em produção em uma VPS (como Hostinger/DigitalOcean).



Gere e estruture os seguintes arquivos e passos organizados para a criação da pasta unificada de infraestrutura:



### 1. MAPEAMENTO DA NOVA PASTA DO REPOSITÓRIO

Crie uma pasta chamada `infra/` na raiz do projeto `mimus/` contendo:

- `infra/docker-compose.yml`: Configuração multi-container isolada em rede interna bridge para rodar o n8n e a Evolution API localmente na VPS. Garanta que o n8n possua chaves de criptografia e variáveis de ambiente explícitas, e que a Evolution API possua volumes persistentes para não perder as sessões do WhatsApp (`evolution_instances`).

- `infra/workflows/`: Diretório destinado aos backups em formato JSON que serão exportados do n8n para manter o histórico de versionamento do Git.



### 2. ARQUIVO DOCKER-COMPOSE.YML (PRONTO PARA PRODUÇÃO / HOSTINGER)

Escreva o código YAML do `docker-compose.yml` completo, configurando:

- Imagem oficial estável do n8n (com portas 5678 expostas).

- Imagem oficial da Evolution API (com portas 8080 expostas).

- Rede interna chamada `mimus_network`.

- Volumes locais para ambos os serviços.



### 3. MANUAL TÉCNICO DE IMPLEMENTAÇÃO E SYSTEM PROMPTS (README.md)

Gere um arquivo `README.md` detalhado para ficar dentro da pasta `infra/`. Este arquivo deve documentar:

- O passo a passo exato de comandos no terminal Linux/Ubuntu para rodar o docker-compose.

- Como acessar e configurar as credenciais do DeepSeek dentro do n8n simulando o nó da OpenAI (Custom URL: `https://api.deepseek.com/v1`).

- Os 5 Prompts de Sistema (System Prompts) cirúrgicos, hiper-estruturados com delimitadores [PAPEL], [DIRETRIZES DE LINGUAGEM] e [DIRETRIZES DE SAÍDA], otimizados para gastar o mínimo de tokens no modelo `deepseek-chat` (V3) e `deepseek-reasoner` (R1) para os seguintes agentes:

  1. SDR / Inside Sales (WhatsApp - linguagem humana, foco em conversão e ativação de leads com gírias leves de estética).

  2. CFO / Financeiro (Análise fria de SaaS: MRR, Churn, CAC, LTV, Burn rate em tabelas Markdown).

  3. Customer Success (Retenção e engajamento ativo de lojistas frias via WhatsApp com dicas de negócios).

  4. Suporte Nível 1 (FAQ operacional amigável baseado estritamente na documentação fornecida).

  5. Core Developer / DevOps (Análise pesada em DeepSeek-R1 de logs de erro em tempo real com entrega de patch/código de correção pronto).
